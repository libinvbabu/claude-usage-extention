# Screenshot plan — Claude Usage Pace

Five screenshot concepts for the Chrome Web Store listing.

## 1. Main dashboard on Claude's Usage page
Shows the top recommendation card, current session card, the two weekly limit cards
side-by-side, the task guidance section, and the local-only footer. The hero shot.

## 2. Current session pacing detail
Close-up on the current session card: large "% left" number, the mini progress bar with
the linear pace marker, the PACE and SAFE PACE metrics, and the "Unused if pace continues"
figure. Hover a tooltip (e.g. Safe pace) to show the explanatory copy.

## 3. Weekly bottleneck state
A state where the session has room but a weekly limit is the bottleneck — top card reads
"Weekly limit is the bottleneck", the bottleneck line names the weekly limit, and the task
guidance shows "Save usage for / Avoid".

## 4. Options page
Working days (Mon–Fri / Mon–Sat / Every day / Custom), working hours, the display toggles
(Compact mode, Show Claude Code tips), and the Privacy explanation block.

## 5. Privacy / local-only reassurance
The footer ("Based on Claude's visible usage bars. Local-only.") together with the Options
Privacy section, to reinforce that nothing leaves the browser.

---

## Capture & polish checklist

- Resolution: 1280×800 (preferred) or 640×400; PNG, no rounded corners, no padding.
- No private account info, organization names, teammate names, or email addresses visible.
- Usage values are plausible (e.g. 31% / 4% / 0% used).
- Parser **debug mode is OFF** for store screenshots (only enable it for beta docs).
- Dashboard centered and readable; crop browser chrome cleanly.
- Use the dark theme (Claude's default) for visual consistency.
- Prefer a fresh profile so no real history/usage is shown.
